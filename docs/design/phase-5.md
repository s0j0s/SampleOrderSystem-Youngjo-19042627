### D-1. 재고 상태 기준 — 주문 대비 판단

단순 재고 임계값이 아니라 **RESERVED 주문 수량 합계**와 비교:

```
stock == 0                          → 고갈 (RED)
0 < stock < reservedQty(sampleId)   → 부족 (YELLOW)
stock >= reservedQty(sampleId)      → 여유 (GREEN)
```

- `reservedQty(sampleId)`: 해당 시료에 대한 RESERVED 상태 주문 수량 합계
- RESERVED 주문이 없는 경우 → stock > 0이면 여유, stock == 0이면 고갈
- 모니터링 화면에서 `List<Order>`를 stream으로 집계: `orders.stream().filter(RESERVED).groupBy(sampleId).sum(quantity)`

### D-2. `readInt()` 버그 수정 — while 루프

```java
public int readInt(String prompt) {
    while (true) {
        System.out.print(prompt);
        try {
            int value = Integer.parseInt(scanner.nextLine().trim());
            if (value > 0) return value;
            showError("양수를 입력하세요.");
        } catch (NumberFormatException e) {
            showError("숫자를 입력하세요.");
        }
    }
}
```

### D-3. `readDouble()` 신규 — yield 전용 (0.0 < val ≤ 1.0)

```java
public double readDouble(String prompt) {
    while (true) {
        System.out.print(prompt);
        try {
            double value = Double.parseDouble(scanner.nextLine().trim());
            if (value > 0.0 && value <= 1.0) return value;
            showError("0.0 초과 1.0 이하의 값을 입력하세요.");
        } catch (NumberFormatException e) {
            showError("숫자를 입력하세요.");
        }
    }
}
```

### D-7. 생산 화면 실시간 갱신 (2~3초 간격)

생산 현황 / 대기 목록 화면에만 적용. 전체 앱 적용은 스레드 안전성 문제로 제외.

**적용 화면**: `showProductionStatus()`, `showProductionQueue()`  
**미적용 화면**: 나머지 모든 화면 (입력 블로킹 구조 유지)

구현 패턴:
```java
while (true) {
    System.out.print("\033[2J\033[H");   // 화면 클리어 (ANSI)
    printHeader("메인 > 생산라인 조회 > 생산 현황");
    renderProductionCards(productions);  // 진행률/남은시간 실시간 계산
    System.out.print("  [Q] 돌아가기  ");

    long deadline = System.currentTimeMillis() + 2_500;
    while (System.currentTimeMillis() < deadline) {
        if (System.in.available() > 0) {
            int ch = System.in.read();
            if (ch == 'q' || ch == 'Q' || ch == '\n') return;
        }
        Thread.sleep(100);
    }
}
```

변경 범위:
- `ConsoleView`: `showProductionStatus()` / `showProductionQueue()` 내부 루프 패턴
- `Main.handleProductionMenu()`: Scanner 대신 위 루프 위임
- ANSI 상수 추가: `private static final String CLEAR = "\033[2J\033[H";`
- 기타 구조 변경 없음

### D-8. 생산 자동 완료 처리

메인 루프 매 반복마다 `productionController.checkAndCompleteExpired()` 호출:
```java
// Main.java while 루프 상단
productionController.checkAndCompleteExpired();
view.showMainMenu(...);
```

`checkAndCompleteExpired()` 로직:
```
pendingProductions.forEach {
    if (now >= startedAt + estimatedHours * 3_600_000) → completeProduction(id)
}
```

**Production 모델 변경 필요 (cross-phase):**  
`startedAt` (long, epoch millis) 필드 추가. `OrderController.approveOrder()` 재고 부족 분기에서 `System.currentTimeMillis()`로 설정.

```
  │  S-003  │  GaAs 기판        │    0  │  ░░░░░░░░░░░░  🔴 고갈            │  ← RED
  └─────────┴──────────────────┴───────┴──────────────────────────────────┘
```

주문 바 차트: `count / max(counts) * 12칸`.  
재고 바 차트: `stock / max(stocks) * 12칸` — 빈 칸은 `░`, 채운 칸은 `█`.  
재고 상태: RESERVED 주문 수량 합계 대비 — 여유/부족/고갈 (D-1 참조).  
부족 상태일 때 `(주문 Nnn개)` 추가 표기.

### 5. 생산라인 조회 서브메뉴

> **생산 완료는 자동 처리**: 예상 시간 경과 시 메인 루프에서 자동으로 CONFIRMED 전환.  
> 수동 완료 메뉴 없음.

```
╔══════════════════════════════════════════════════════╗
║  🔬 S-SEMI  │  메인 > 생산라인 조회                    ║
╚══════════════════════════════════════════════════════╝

  ┌──────────────────────────────────┐
  │  [1] 생산 현황                    │
  │  [2] 대기 목록 (FIFO 큐)          │
  │  [0] ← 돌아가기                   │
  └──────────────────────────────────┘

  선택> 
```

```
  │  진행률  ███░░░░░░░  30%   남은 시간  18.9h          │

  └─────────────────────────────────────────────────────┘
```

| 계산 항목 | 공식 |
|-----------|------|
| 부족분 | `max(0, 주문수량 - 승인시점재고)` — Production 생성 시점에 고정 저장 |
| 실 생산량 | `ceil(부족분 / (수율 × 0.9))` |
| 총 생산 시간 | `단위생산시간 × 실생산량` |
| elapsed | `(now - startedAt) / 3_600_000.0` (h) |
| 진행률 | `min(1.0, elapsed / estimatedHours) × 10칸` |
| 현재 생산량 | `(int)(진행률 × productionQty)` |
| 완료 예정 | `startedAt + estimatedHours * 3_600_000` → LocalDateTime 포맷 |

`showMonitor`에서 재고 상태 판단에 사용하는 **현재 재고**와 구분:  
생산 현황 카드의 "현재 재고"는 **승인 시점 재고** (Production 생성 시 snapshot 저장).

```java
public int readListChoice(int max) {
    while (true) {
```

    while (true) {

### 5-2. `Main.java` — 메뉴 루프 완성

```java
switch (choice) {
    case 1 -> handleSampleMenu(sampleController, view);
    case 2 -> handleOrderCreate(orderController, sampleController, view);
    case 3 -> handleOrderApproval(orderController, view);
    case 4 -> handleMonitor(orderController, sampleController, view);
    case 5 -> handleProductionMenu(productionController, view);
    case 6 -> handleRelease(orderController, view);
    case 7 -> { view.showInfo("시스템을 종료합니다."); running = false; }
    default -> view.showError("유효하지 않은 메뉴입니다.");
}
```

**핸들러 6개:**

| 메서드 | 동작 |
|--------|------|
| `handleSampleMenu` | 서브메뉴 switch: 등록/조회/검색 |
| `handleOrderCreate` | 시료 목록 출력 → 번호 선택 → createOrder |
| `handleOrderApproval` | 서브메뉴 switch: 목록조회 / 주문처리(번호선택→승인·거절) |
| `handleMonitor` | showMonitor 직접 호출 |
| `handleProductionMenu` | 서브메뉴 switch: 생산현황(진행률/현재량) / 대기목록(FIFO) |
| `handleRelease` | CONFIRMED 목록 번호 선택 → 확인 → releaseOrder |

**번호 선택 흐름 (승인/거절/생산완료/출고):**
1. 목록 출력 (비어있으면 안내 후 즉시 반환)
2. `view.readListChoice(list.size())` → 0이면 복귀
3. 선택된 항목 상세 출력 (거절/생산완료/출고만)
4. `view.readConfirm(...)` → n이면 취소 메시지 후 복귀
5. Controller 처리 → 결과 출력

모든 핸들러: `IllegalArgumentException` / `IllegalStateException` → `view.showError(e.getMessage())`

| 주문 승인(재고 부족) | PRODUCING + PRD 번호 출력 |
| 모니터링 | REJECTED 제외 4종 카운트, 재고 여유/부족/고갈 상태, 부족 시 주문량 표기 |
| 생산 현황 | 진행률 바 차트, 현재 생산량, 남은 시간 표시 |
| 대기 목록 | FIFO 순서, 시작 시각 표시 |
| 자동 완료 | 예상 시간 경과 후 메인 복귀 시 CONFIRMED 자동 전환, 잉여재고 입고 확인 |
| 출고 관리 | CONFIRMED 목록 선출력, RELEASE 완료 메시지 |
